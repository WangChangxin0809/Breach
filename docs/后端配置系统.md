# 后端配置系统

后端玩法参数已从 Go 常量迁移为 **JSON 配置 + Go 强类型加载 + 启动校验**。

## 配置文件

默认配置位于：

- `server/modules/config/data/match.json`：对局参数、地图尺寸、障碍物、击杀奖励。
- `server/modules/config/data/characters.json`：角色血量、速度、复活时间、视野、碰撞半径。
- `server/modules/config/data/weapons.json`：武器伤害、射速、射程、弹容量、解锁费用。

这些 JSON 会通过 `go:embed` 打进 Nakama Go 插件。服务启动时，`server/modules/config/config.go` 会加载并校验配置，然后通过 `config.Active()` 提供给游戏逻辑使用。

## 如何调数

直接修改对应 JSON，然后重建/重启 Nakama 服务。

示例：调整默认角色生命值：

```json
{
  "id": "recruit",
  "base_health": 120,
  "max_speed": 220.0
}
```

如果配置非法，例如 tick rate 超出范围、角色血量小于等于 0、默认角色不存在，服务会在启动时失败，避免带着错误配置进入对局。

## 如何新增角色

在 `characters.json` 的 `characters` 数组中添加角色：

```json
{
  "id": "scout",
  "base_health": 85,
  "max_speed": 260.0,
  "respawn_delay_sec": 5,
  "collision_radius": 16.0,
  "vision_radius": 170.0,
  "vision_cone_length": 450.0
}
```

后续选人系统只需要把 `Player.CharacterID` 设置为对应 ID，例如 `scout`。移动校验会按该角色读取速度和碰撞半径；找不到角色 ID 时会回退到默认角色。

## 如何新增武器

在 `weapons.json` 的 `weapons` 数组中添加武器：

```json
{
  "id": "rifle_m4",
  "slot": "primary",
  "damage": 32,
  "fire_rate": 9.0,
  "range": 820.0,
  "ammo_capacity": 30,
  "unlock_cost": 300
}
```

当前武器配置结构已经准备好，后续射击、经济、解锁系统应通过 `config.Active().Weapons` 读取这些数据。

## 外部配置覆盖

默认使用内嵌配置。需要在本地或服务器覆盖配置时，设置环境变量：

```bash
BREACH_CONFIG_DIR=/path/to/config-data
```

该目录必须包含：

```text
match.json
characters.json
weapons.json
```

## 开发规则

- 可调玩法参数放入 JSON，不再新增 `PLAYER_MAX_SPEED`、`ROUND_DURATION_SEC` 这类 Go gameplay 常量。
- 新增 JSON 字段时，需要同步更新 `config.go` 中的结构体和 `Validate()` 校验。
- 游戏逻辑通过 `config.Active()` 或传入的 `*config.GameConfig` 读取配置。
- 稳定技术标识仍可使用 Go 常量，例如 match module name、op code、协议版本。
