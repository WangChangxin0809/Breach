# 新增角色与武器规范

本文是添加新角色或新武器时的简明 checklist。原则是：客户端负责表现和操作体验，服务端负责权威数值和校验。只新增客户端场景是不完整的，角色/武器的基础信息通常也必须同步更新后端配置。

## 新增角色

### 1. 客户端表现

新增角色至少需要准备这些内容：

- 角色贴图或动画资源，放在 `client/assets/spiritframes/` 或后续统一角色资源目录。
- 一个角色场景，建议从 `client/scenes/characters/player_character.tscn` 复制或继承。
- 一个具体角色脚本，继承 `client/scripts/characters/character_base.gd`。如果没有特殊逻辑，可以先只保留空壳或少量导出属性。
- 在角色场景中配置：
  - `AnimatedSprite2D`，包含至少 Idle 动画；如果角色可移动，应包含 Run 动画。
  - `CollisionShape2D`。
  - 可选的身体灯光节点。
  - `WeaponPivot`，挂载默认武器场景。
  - 如有技能，放到 `AbilitySlot/Basic` 和 `AbilitySlot/Ultimate`。

角色场景需要保持以下约定：

- 根节点建议是 `CharacterBody2D`。
- 根节点脚本必须继承 `CharacterBase`。
- 动画节点默认命名为 `AnimatedSprite2D`，否则需要设置 `sprite_node_path`。
- 武器挂点默认命名为 `WeaponPivot`，否则需要设置 `weapon_node_path`。
- Idle/Run 动画名称如果不是 `Idle` / `Run`，需要设置 `idle_animation` / `run_animation`。

### 2. 选人界面

如果这个角色要出现在当前选人界面，需要更新：

- `client/scripts/main_game/character_select.gd`

在 `characters` 数组中添加一项：

```gdscript
{
	"id": "new_character_id",
	"name": "显示名",
	"role": "定位说明",
	"skill": "基础技能名",
	"ultimate": "终极技能名",
	"portrait": preload("res://assets/spiritframes/xxx.png"),
}
```

注意：这里的 `id` 必须和后端 `characters.json` 中的角色 `id` 完全一致。客户端锁定角色时会把这个 ID 发给服务端。

### 3. 后端权威配置

必须更新：

- `server/modules/config/data/characters.json`

添加角色配置：

```json
{
  "id": "new_character_id",
  "base_health": 100,
  "max_speed": 220.0,
  "respawn_delay_sec": 5,
  "collision_radius": 16.0,
  "vision_radius": 180.0,
  "vision_cone_length": 420.0
}
```

这些字段是服务端权威值：

- `base_health`：基础生命值。
- `max_speed`：服务端移动速度校验使用的最大速度。
- `respawn_delay_sec`：复活延迟。
- `collision_radius`：服务端移动碰撞半径。
- `vision_radius`：近距离圆形视野半径。
- `vision_cone_length`：前方锥形视野长度。

如果只在客户端添加角色，但没有更新 `characters.json`，服务端收到该角色 ID 后会回退到默认角色配置，导致表现和权威数值不一致。

### 4. 角色新增检查

完成后至少检查：

- 角色场景能单独实例化。
- 角色进入 `main` 后能被网络视觉模板复制。
- 移动时 Idle/Run 动画切换正常。
- 武器会跟随朝向旋转和翻转。
- 选人界面的角色 ID 与后端 `characters.json` 一致。
- `go vet ./modules/...` 在 `server/` 目录下通过。

## 新增武器

### 1. 客户端表现

新增武器至少需要准备这些内容：

- 武器贴图，放在 `client/assets/weapon/` 或后续统一武器资源目录。
- 一个武器场景，建议从 `client/scenes/weapons/sidearm_gun.tscn` 复制，或继承 `client/scenes/weapons/weapon_base.tscn`。
- 如有特殊表现，再新增脚本继承 `client/scripts/weapons/weapon_base.gd`。

武器场景需要保持以下约定：

- 根节点建议继承/实例化 `weapon_base.tscn`。
- 根节点脚本必须继承 `WeaponBase`。
- 武器贴图节点默认命名为 `WeaponSprite`，否则需要设置 `sprite_node_path`。
- 设置 `weapon_id`，用于客户端识别。
- 按需要添加 `PointLight2D`、枪口特效挂点、音效挂点等子节点。

### 2. 角色装配

如果新武器要成为某个角色的默认武器：

- 打开对应角色场景，例如 `client/scenes/characters/player_character.tscn`。
- 把 `WeaponPivot` 替换为新武器场景实例。
- 确保位置偏移、贴图方向、灯光位置正确。

如果新武器需要运行时切换：

- 使用 `CharacterBase.equip_weapon(weapon_scene)` 替换当前武器。
- 后续背包/装备系统应根据服务端下发或本地已解锁状态选择对应武器场景。

### 3. 后端权威配置

通常必须更新：

- `server/modules/config/data/weapons.json`

添加武器配置：

```json
{
  "id": "new_weapon_id",
  "slot": "primary",
  "damage": 28,
  "fire_rate": 8.0,
  "range": 760.0,
  "ammo_capacity": 30,
  "unlock_cost": 0
}
```

这些字段是服务端权威值：

- `slot`：装备槽位，例如 `sidearm`、`primary`、`attachment`、`utility`。
- `damage`：伤害。
- `fire_rate`：射速。
- `range`：射程。
- `ammo_capacity`：弹容量。
- `unlock_cost`：解锁成本。

客户端武器场景的 `weapon_id` 应尽量和后端 `weapons.json` 的 `id` 保持一致。否则后续装备、解锁、射击校验、UI 显示会很容易出现映射错误。

### 4. 武器新增检查

完成后至少检查：

- 武器场景能单独实例化。
- 武器挂到角色后会跟随角色朝向旋转。
- 鼠标在角色左右两侧时，武器贴图翻转正确。
- 武器灯光会随阵营色变化。
- `weapon_id` 与后端 `weapons.json` 中的 `id` 一致。
- `go vet ./modules/...` 在 `server/` 目录下通过。

## 必须同步后端吗？

结论：如果新增内容会进入正式联机玩法，必须同步后端配置。

可以只改客户端的情况：

- 临时美术预览。
- 不参与联机的本地测试场景。
- 只替换贴图、灯光、动画等纯表现资源，且角色/武器 ID 不变。

必须同步后端的情况：

- 新角色会被选人系统锁定并进入比赛。
- 新角色有不同生命值、速度、碰撞半径、视野或复活时间。
- 新武器会参与射击、伤害、弹药、解锁、购买或装备。
- 新武器有新的 ID、槽位、伤害、射速、射程、弹容量或解锁成本。

后端配置是权威来源。客户端表现可以先行占位，但最终必须让客户端 ID、选人/装备逻辑、服务端 JSON 配置保持一致。

## 推荐提交粒度

新增一个完整角色时，建议同一 PR 至少包含：

- 客户端角色资源。
- 客户端角色场景和脚本。
- 选人界面条目。
- `server/modules/config/data/characters.json` 配置。
- 基础 Godot probe 或手动验证说明。

新增一个完整武器时，建议同一 PR 至少包含：

- 客户端武器资源。
- 客户端武器场景和脚本。
- 角色默认装配或装备映射。
- `server/modules/config/data/weapons.json` 配置。
- 基础 Godot probe 或手动验证说明。
