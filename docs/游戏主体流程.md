# 游戏主体流程

## 目标

游戏主流程需要把登录、组队、匹配、对局和结算拆清楚，避免大厅逻辑、匹配逻辑和战斗逻辑互相耦合。核心原则是：

- 大厅负责玩家状态、社交、装备与入口选择。
- 队伍房间负责组队、邀请、准备和发起匹配。
- 匹配系统负责凑齐玩家并创建服务器权威对局。
- 对局内只处理实时战斗、回合规则和服务器权威状态同步。
- 结算页负责展示结果、奖励和下一步去向。

## 主流程概览

```text
启动游戏
  -> 自动登录 / 登录注册
  -> 主大厅
    -> 快速匹配
    -> 多人组队
      -> 队伍房间
      -> 开始匹配
    -> 自定义房间
    -> 训练模式
  -> 匹配成功
  -> 赛前准备
  -> 正式对局
  -> 赛后结算
  -> 返回大厅 / 再来一局
```

## 详细流程

### 1. 启动游戏

客户端启动后先加载本地配置、初始化网络模块，并尝试连接 Nakama。

如果本地存在有效 session，则自动登录并进入主大厅；如果 session 失效或不存在，则进入登录页。

### 2. 登录 / 注册

登录方式可以分阶段实现：

- MVP 阶段：游客登录。
- 后续阶段：账号密码登录、第三方登录、游客账号绑定。

登录成功后，客户端拉取玩家基础信息、好友状态、装备配置和必要的服务端配置，然后进入主大厅。

### 3. 主大厅

主大厅是玩家的默认停留界面，负责展示玩家信息和主要功能入口。

建议包含：

- 玩家信息：昵称、头像、等级、货币、段位或战绩摘要。
- 游戏入口：快速匹配、多人组队、自定义房间、训练模式。
- 准备入口：角色、装备、背包、设置。
- 社交入口：好友列表、邀请、队伍状态。

### 4. 选择游戏模式

推荐模式入口如下：

- 快速匹配：直接进入默认 3v3 爆破匹配，适合单人快速开局。
- 多人组队：先进入队伍房间，可邀请好友，再由队长开始匹配。
- 自定义房间：创建私人房间，可设置地图、回合数、密码等。
- 训练模式：单人练枪、熟悉地图、测试武器与技能。

MVP 阶段优先实现快速匹配和多人组队；自定义房间、训练模式可以后置。

### 5. 多人组队房间

多人组队房间用于匹配前的队伍管理。

基础能力：

- 房主创建队伍。
- 邀请好友加入。
- 队员选择角色和装备。
- 队员切换准备状态。
- 房主开始匹配。

扩展能力：

- 踢出队员。
- 转让房主。
- 队内聊天。
- 显示队员延迟、在线状态和加载状态。

队伍未满 3 人时也可以开始匹配，由系统补齐队友。

### 6. 匹配阶段

匹配系统根据队伍人数、玩家水平、地区和延迟寻找对手。

找到 6 名玩家后进入确认阶段。所有玩家确认后，由服务端创建权威 match 并通知客户端进入加载流程。

如果有人拒绝或超时，则将已确认玩家送回匹配队列或队伍房间，具体策略可后续调整。

### 7. 赛前准备

赛前准备发生在匹配成功之后、正式回合开始之前。

需要完成：

- 分配攻击方和防守方。
- 加载地图和服务端下发的对局配置。
- 确认玩家角色和初始装备。
- 等待所有客户端加载完成。
- 进入开局倒计时。

倒计时结束后，客户端进入正式对局界面，服务端开始权威 MatchLoop。

### 8. 正式对局

对局由服务端权威运行，客户端只发送输入和操作请求。

每回合流程：

```text
回合开始
  -> 玩家出生
  -> 战斗 / 安包 / 拆包 / 击杀中立目标
  -> 回合结束判定
  -> 回合结算
  -> 下一回合
```

半场结束后双方换边，并重置需要重置的经济或积分数据。整场比赛结束后进入赛后结算。

### 9. 赛后结算

结算页展示本局结果和玩家表现。

建议包含：

- 胜负结果。
- 回合比分。
- 击杀、死亡、助攻、伤害、安包、拆包等数据。
- MVP 或关键表现。
- 经验、货币、任务进度等奖励。

结算后的去向：

- 再来一局。
- 返回队伍房间。
- 返回主大厅。

## MVP 优先级

第一阶段建议只实现最短闭环：

```text
游客自动登录
  -> 主大厅
  -> 快速匹配 / 创建队伍
  -> 匹配成功
  -> 进入对局
  -> 赛后结算
```

好友邀请、自定义房间、战绩、任务、段位和复杂奖励系统可以后续逐步加入。这样可以先验证联网、匹配、权威对局和结算闭环，再扩展外围系统。
