## 安装 Docker

### macOS

#### 方法一：Docker Desktop（推荐）

1. 访问 [Docker Desktop 官网](https://www.docker.com/products/docker-desktop)

2. 下载适用于 Mac 的 Docker Desktop

3. 双击 `Docker.dmg` 安装包

4. 将 Docker.app 拖入应用程序文件夹

5. 启动 Docker Desktop（首次启动需要几分钟初始化）

6. 验证安装：
```bash
docker --version
docker-compose --version
```

#### 方法二：Homebrew

```bash
brew install --cask docker
```

### Windows

1. 访问 [Docker Desktop 官网](https://www.docker.com/products/docker-desktop)

2. 下载适用于 Windows 的 Docker Desktop

3. 双击 `Docker Desktop Installer.exe` 运行安装程序

4. 安装 WSL 2（如果提示）

5. 启动 Docker Desktop

6. 验证安装：
```powershell
docker --version
docker-compose --version
```

### Linux (Ubuntu/Debian)

```bash
# 卸载旧版本
sudo apt-get remove docker docker-engine docker.io containerd runc

# 安装依赖
sudo apt-get update
sudo apt-get install \
    apt-transport-https \
    ca-certificates \
    curl \
    gnupg \
    lsb-release

# 添加 Docker GPG 密钥
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

# 添加 Docker 仓库
echo \
  "deb [arch=amd64 signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 安装 Docker
sudo apt-get update
sudo apt-get install docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 将当前用户加入 docker 用户组（免 sudo）
sudo usermod -aG docker $USER

# 重新登录或重启生效
```

## 配置 Nakama 后端

### 1. 进入后端目录

```bash
cd <repo-root>/server
```

### 2. 目录结构说明

```
server/
├── docker-compose.yml    # Docker 容器编排配置
├── Dockerfile           # Nakama 镜像构建配置
├── local.yml            # Nakama 本地配置文件
├── go.mod               # Go 模块定义（module breach3v3/server）
├── modules/             # 游戏逻辑代码
│   ├── config/          # 配置参数
│   ├── state/           # 游戏状态
│   ├── systems/         # 游戏系统
│   ├── match_handler.go # 匹配处理器
│   └── protobuf/        # Protobuf 定义
```

### 3. 配置文件说明

#### docker-compose.yml 主要端口

| 端口 | 服务 | 说明 |
|------|------|------|
| 7349 | Nakama | gRPC 端口 |
| 7350 | Nakama | WebSocket 端口（游戏连接用） |
| 7351 | Nakama Console | Web 管理界面 |

#### 数据库配置

- 数据库名：`nakama`
- 用户名：`postgres`
- 密码：`localdb`
- 地址：`postgres:5432/nakama`

### 4. 构建自定义 Nakama 镜像

如果需要使用自定义的 Go 代码构建 Nakama 镜像：

```bash
cd server/

# 构建镜像（包含自定义模块）
docker-compose build

# 或只构建 Nakama
docker-compose build nakama
```

## 启动本地服务器

### 1. 启动所有服务

```bash
cd server/

# 启动 Nakama + PostgreSQL
docker-compose up -d
```

### 2. 查看服务状态

```bash
# 查看运行状态
docker-compose ps

# 查看日志
docker-compose logs -f nakama

# 查看 Nakama 实时日志
docker logs -f breach3v3-nakama
```

### 3. 验证服务

#### 检查 Nakama 健康状态

```bash
curl http://localhost:7351/healthcheck
```

预期输出：
```json
{"alive": true}
```

#### 检查 WebSocket 连接

在浏览器中打开 Godot 编辑器或使用测试工具连接 `ws://localhost:7350`

### 4. 访问 Nakama Console

1. 打开浏览器访问：http://localhost:7351

2. 默认登录凭证：
   - 用户名：`admin`
   - 密码：`password`

3. Console 功能：
   - 查看在线用户
   - 查看活跃匹配
   - 管理用户账户
   - 查看服务器状态

### 5. 停止服务

```bash
cd server/

# 停止所有服务（保留数据）
docker-compose stop

# 停止并删除容器（保留数据卷）
docker-compose down

# 停止并删除所有数据（完全清理）
docker-compose down -v
```

## 运行 Godot 客户端

### 1. 安装 Godot

1. 访问 [Godot 官网](https://godotengine.org/download)

2. 下载 Godot 4.6.2（推荐）

3. macOS：将 Godot.app 放入应用程序文件夹

4. Windows：解压到指定目录

### 2. 打开项目

```bash
# 使用命令行启动 Godot（macOS/Linux）
/Applications/Godot.app/Contents/MacOS/Godot --path <repo-root>/client/

# Windows
# "C:\Program Files\Godot\Godot.exe" --path "C:\path\to\Breach-3v3"
```

或直接在 Godot 编辑器中：
1. 点击"导入"
2. 选择 `client/project.godot` 文件（**注意：项目根在 `client/` 子目录，不是仓库根**）

### 3. 运行游戏

1. 在 Godot 编辑器中点击"运行"（▶️ 按钮）

2. 或按 `Cmd/Ctrl + R`

### 4. 多客户端测试（Godot 编辑器方式）

需要测试多人游戏时，Godot 4.x 提供了便捷的多实例运行功能：

#### 配置多实例运行

1. 在 Godot 编辑器中，点击顶部菜单 **调试**
2. 选择 **自定义运行实例**
3. 勾选 **启用多实例**
4. 将 **数量** 设置为 `2`
5. 点击 **关闭** 保存设置

#### 启动测试

1. 在 Godot 编辑器中点击"运行"（▶️ 按钮）

2. 编辑器会自动启动 **2 个游戏窗口**，分别作为客户端 1 和客户端 2

3. 两个窗口可以分别：
   - 登录不同的测试账户
   - 连接到同一个本地游戏服务器
   - 验证多人同步功能

#### 验证清单

- [ ] 两个窗口都能正常启动
- [ ] 两个客户端都能连接到 Nakama 服务器
- [ ] 在一个窗口移动时，另一个窗口能看到同步的位置变化
- [ ] 服务器控制台显示两个客户端的连接日志

### 5. 命令行多实例方式

如果需要更多灵活性，可以使用命令行启动多个实例：

1. 启动第一个 Godot 实例作为客户端 1

2. 启动第二个 Godot 实例作为客户端 2
   ```bash
   # macOS 终端方式启动第二个实例
   /Applications/Godot.app/Contents/MacOS/Godot --path <repo-root>/client/
   ```

3. 两个客户端都会连接到本地 Nakama 服务器进行测试

## 常见问题

### Q1: Docker 启动失败

**症状**：执行 `docker-compose up` 报错

**解决方案**：
```bash
# 重启 Docker Desktop
killall Docker && open -a Docker

# 清理并重试
docker-compose down
docker system prune -f
docker-compose up -d
```

### Q2: 端口被占用

**症状**：`port is already allocated`

**解决方案**：
```bash
# 查看哪个进程占用了端口
# macOS/Linux
lsof -i :7350

# Windows
netstat -ano | findstr 7350

# 停止占用进程或修改 docker-compose.yml 中的端口映射
```

### Q3: 数据库连接失败

**症状**：`could not connect to database`

**解决方案**：
```bash
# 等待 PostgreSQL 完全启动
docker-compose logs postgres

# 重启服务
docker-compose restart
```

### Q4: Go 模块编译失败

**症状**：`go build` 报错

**解决方案**：
```bash
cd server/
go mod download
go mod tidy
go build ./...
```

### Q5: Godot 无法连接服务器

**检查项**：
1. 确认 Nakama 容器正在运行：`docker-compose ps`
2. 确认端口映射正确：`docker port breach3v3-nakama`
3. 检查防火墙设置
4. 查看 Godot 控制台输出的错误信息

### Q6: 容器内存不足

**解决方案**：

Docker Desktop 设置：
- macOS：Docker Desktop → Preferences → Resources → 调整内存到 4GB+
- Windows：Docker Desktop → Settings → Resources → 调整内存到 4GB+
