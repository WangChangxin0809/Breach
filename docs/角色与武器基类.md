# 角色与武器基类

本文说明当前客户端角色/武器抽象的最小设计。目标是让后续新增角色或武器时，优先通过继承基础场景和配置节点属性完成，而不是在 `main.gd` 或地图场景中重复写移动、动画、朝向和灯光逻辑。

## 角色基类

角色基类由脚本 `client/scripts/characters/character_base.gd` 和场景 `client/scenes/characters/character_base.tscn` 组成。

`CharacterBase` 继承 `CharacterBody2D`，负责所有角色共用能力：

- 本地移动与碰撞推动。
- Idle/Run 动画切换。
- 角色朝向、左右翻转。
- 武器挂载与 `equip_weapon()`。
- 血量、死亡、复活信号。
- 服务器权威状态应用。
- 角色身体颜色、灯光显示和灯光阵营色。

核心导出属性：

- `local_movement_enabled`：是否启用本地输入移动。网络视觉复制体会关闭它。
- `move_speed`：客户端本地预览/测试移动速度。
- `sprite_node_path`：角色动画节点路径，默认 `AnimatedSprite2D`。
- `weapon_node_path`：武器挂载节点路径，默认 `WeaponPivot`。
- `idle_animation` / `run_animation`：动画名称，默认 `Idle` / `Run`。
- `use_custom_cursor` / `custom_cursor`：具体玩家角色可启用自定义鼠标光标。

主要方法：

- `set_facing(facing: Vector2)`：设置角色朝向，并同步武器旋转与翻转。
- `face_position(world_position: Vector2)`：让角色朝向世界坐标。
- `set_moving(moving: bool)`：切换 Idle/Run 动画。
- `set_body_modulate(color: Color)`：设置角色身体颜色。
- `set_lights_visible(enabled: bool)`：统一控制角色下所有 `PointLight2D` 可见性。
- `set_light_color(color: Color)`：统一设置角色下所有 `PointLight2D` 颜色。
- `equip_weapon(weapon_scene: PackedScene)`：替换当前武器挂载。
- `apply_authoritative_state(state: Dictionary)`：应用服务器广播的基础状态。

## 武器基类

武器基类由脚本 `client/scripts/weapons/weapon_base.gd` 和场景 `client/scenes/weapons/weapon_base.tscn` 组成。

`WeaponBase` 继承 `Node2D`，负责所有武器共用的表现逻辑：

- 根据角色朝向旋转武器。
- 根据左右方向翻转武器贴图。
- 保持武器 pivot 的左右偏移。
- 控制武器灯光显示和灯光颜色。
- 支持运行时替换武器贴图。

核心导出属性：

- `weapon_id`：武器技术 ID，当前仅用于客户端识别。
- `sprite_node_path`：武器贴图节点路径，默认 `WeaponSprite`。
- `base_x_offset`：武器在角色左右两侧的基础偏移。小于 `0` 时会在 `_ready()` 中使用当前 `position.x` 的绝对值。

主要方法：

- `set_facing(facing: Vector2)`：旋转武器并处理左右翻转。
- `set_lights_visible(enabled: bool)`：控制武器下所有 `PointLight2D` 可见性。
- `set_light_color(color: Color)`：设置武器下所有 `PointLight2D` 颜色。
- `set_weapon_texture(texture: Texture2D)`：替换武器贴图。

## 当前测试角色

当前测试角色场景是 `client/scenes/characters/player_character.tscn`，脚本是 `client/scripts/characters/Player.gd`。

`Player.gd` 现在是一个很薄的具体角色类：

- 继承 `CharacterBase`。
- 只保留当前玩家角色特有的鼠标光标设置。
- 不再重复实现移动、动画切换、武器旋转和翻转逻辑。

`player_character.tscn` 内部包含：

- `AnimatedSprite2D`：使用当前 Player_01 的 `Idle` 和 `Run` 动画。
- `CollisionShape2D`：角色碰撞形状。
- `BodyLight`：角色身体灯光。
- `WeaponPivot`：实例化当前测试武器 `sidearm_gun.tscn`。
- `Camera2D`：单独打开角色场景或美术测试时使用；进入 `main` 网络视觉复制时会被 `main.gd` 移除。

`client/scenes/test/art_test.tscn` 中的 `Player` 现在实例化 `player_character.tscn`，不再在地图场景里手写完整角色节点树。

## 当前测试武器

当前测试武器场景是 `client/scenes/weapons/sidearm_gun.tscn`。

它继承 `client/scenes/weapons/weapon_base.tscn`，只定义当前手枪的具体表现：

- `weapon_id = "default_sidearm"`。
- `WeaponSprite.texture = res://assets/weapon/gun.png`。
- 一个 `PointLight2D` 作为武器灯光。

武器朝向、左右翻转、灯光开关和灯光染色都来自 `WeaponBase`，不再散落在角色脚本或 `main.gd` 里。

## 角色/武器注册表

当前主场景通过两个注册表把技术 ID 映射到具体客户端场景：

- `client/scripts/main_game/character_registry.gd`：提供选人展示信息、角色场景和角色默认武器 ID。
- `client/scripts/main_game/weapon_registry.gd`：提供武器场景和武器展示信息。

服务端广播的 `PlayerState.character_id` 会被客户端解码后交给 `main.gd`。`main.gd` 根据 `character_id` 从 `CharacterRegistry` 实例化角色视觉体，再根据默认武器 ID 从 `WeaponRegistry` 装备武器。

现阶段 `fura`、`isaac`、`morgan` 和默认 `recruit` 都复用 `player_character.tscn`；这是为了保持当前测试资源不扩散。后续新增真实角色时，只需要在注册表里把新 ID 指向新的角色场景。

## main 场景中的使用方式

`client/scripts/main_game/main.gd` 会优先根据 `character_id` 从 `CharacterRegistry` 创建网络视觉角色；如果场景查找失败，才回退到 `ArtWorld/Player` 模板。

复制体会保留角色脚本和武器脚本，但会做两件事：

- 关闭 `local_movement_enabled`，避免网络复制体读取本地输入。
- 移除 `Camera2D`，避免每个网络角色都带相机。

视觉同步时，`main.gd` 会优先调用基类 API：

- 朝向：`set_facing()`。
- 动画：`set_moving()`。
- 身体颜色：`set_body_modulate()`。
- 灯光显示：`set_lights_visible()`。
- 灯光颜色：`set_light_color()`。

如果某个临时旧节点没有这些方法，`main.gd` 仍保留旧的节点查找 fallback，方便逐步迁移。

## 新增角色流程

1. 新建角色场景，建议从 `player_character.tscn` 或 `character_base.tscn` 继承/复制。
2. 挂载继承 `CharacterBase` 的具体角色脚本。
3. 配置 `AnimatedSprite2D`、碰撞形状、身体灯光。
4. 配置 `idle_animation`、`run_animation`、`move_speed` 等导出属性。
5. 在 `WeaponPivot` 位置实例化一个武器场景。
6. 如有角色专属技能，把节点挂到 `AbilitySlot/Basic` 和 `AbilitySlot/Ultimate`。
7. 在 `character_registry.gd` 中添加角色 ID、展示信息、场景和默认武器 ID 映射。

新增角色脚本只应写角色差异，例如特殊光标、技能入口、特殊动画名称或角色专属表现。通用移动、朝向、武器翻转、血量和灯光逻辑应留在 `CharacterBase`。

## 新增武器流程

1. 新建武器场景，建议继承/复制 `weapon_base.tscn`。
2. 设置 `weapon_id`。
3. 设置 `WeaponSprite.texture`。
4. 按需要添加枪口灯、特效挂点、音效挂点等子节点。
5. 如果只是表现差异，不需要写新脚本；如果有特殊表现，再写脚本继承 `WeaponBase`。
6. 在 `weapon_registry.gd` 中添加武器 ID 到场景的映射。
7. 在角色场景中把 `WeaponPivot` 替换为新武器场景，或运行时调用 `equip_weapon()`。

当前抽象只处理客户端表现层。伤害、射速、射程、解锁费用等服务器权威数据仍应放在 `server/modules/config/data/weapons.json`，角色血量、速度、视野等权威数据仍应放在 `server/modules/config/data/characters.json`。
