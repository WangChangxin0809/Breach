# 角色武器注册表重构

本文记录当前这次“小而稳”的角色/武器重构。目标是先把 ID 到客户端场景的映射集中起来，让后续新增角色或武器时少改主场景、少复制逻辑。

## 当前结构

客户端新增两个轻量注册表：

- `client/scripts/main_game/character_registry.gd`
- `client/scripts/main_game/weapon_registry.gd`

`CharacterRegistry` 负责：

- 提供选人界面使用的角色展示信息。
- 根据 `character_id` 返回角色场景。
- 根据 `character_id` 返回默认武器 ID。
- 对未知角色 ID 回退到 `recruit`。

`WeaponRegistry` 负责：

- 根据 `weapon_id` 返回武器场景。
- 提供武器展示信息。
- 对未知武器 ID 回退到 `default_sidearm`。
- 临时兼容旧 ID `sidearm_gun`，但当前权威 ID 是 `default_sidearm`。

## 运行时数据流

1. 选人界面从 `CharacterRegistry.selectable_characters()` 读取当前可选角色。
2. 客户端锁定角色后，把角色 ID 通过 `CharacterSelect.character_id` 发给服务端。
3. 服务端用 `server/modules/config/data/characters.json` 查找角色 ID；未知 ID 会回退到默认角色，并把最终角色 ID 写入 `state.Player.CharacterID`。
4. 服务端广播 `GameState.PlayerState.character_id`。
5. 客户端 `ProtobufCodec.decode_player_state()` 解出 `character_id`。
6. `main.gd` 用 `CharacterRegistry.character_scene(character_id)` 创建网络角色视觉体。
7. `main.gd` 用 `CharacterRegistry.default_weapon_id(character_id)` 和 `WeaponRegistry.weapon_scene(weapon_id)` 装备默认武器。
8. 如果同一个玩家的 `character_id` 变化，`main.gd` 会重建该玩家视觉体，避免旧角色场景继续复用。

## 当前测试角色

当前可选角色仍是：

- `fura`
- `isaac`
- `morgan`

当前默认角色是：

- `recruit`

现阶段它们都映射到同一个测试角色场景：

- `client/scenes/characters/player_character.tscn`

这样做是有意保持改动小：先打通“角色 ID -> 场景 -> 武器”的扩展路径，不急着一次性拆出三套不同角色资源。以后新增真实角色时，只需要在注册表中把对应 ID 指向新的角色场景。

## 当前测试武器

当前默认武器 ID 是：

- `default_sidearm`

它映射到测试手枪场景：

- `client/scenes/weapons/sidearm_gun.tscn`

`sidearm_gun.tscn` 继承 `weapon_base.tscn`，只负责具体贴图、灯光和 `weapon_id`。武器朝向、左右翻转、灯光颜色等共用表现仍由 `WeaponBase` 处理。

## 新增角色时改哪里

新增一个会进入联机玩法的角色，至少需要同步这些位置：

- 后端：在 `server/modules/config/data/characters.json` 添加同名 `id` 和权威数值。
- 客户端：新增角色场景，根脚本继承 `CharacterBase`。
- 客户端：在 `client/scripts/main_game/character_registry.gd` 添加角色展示信息、场景映射和默认武器 ID。
- 客户端：如果角色数量超过当前选人界面的 3 个卡片，需要同步扩展选人 UI。

角色 ID 必须保持一致。客户端可以先做纯表现预览，但只要角色会被选人系统发送到服务端，后端 `characters.json` 就必须有对应配置。

## 新增武器时改哪里

新增一个会进入联机玩法的武器，至少需要同步这些位置：

- 后端：在 `server/modules/config/data/weapons.json` 添加同名 `id` 和权威数值。
- 客户端：新增武器场景，建议继承 `weapon_base.tscn`。
- 客户端：在 `client/scripts/main_game/weapon_registry.gd` 添加武器 ID 到场景的映射。
- 客户端：如果它是某个角色默认武器，在 `CharacterRegistry` 的对应角色条目里返回该武器 ID。

武器 ID 也应与后端一致。客户端武器场景负责表现，伤害、射速、射程、弹药和解锁费用仍以后端配置为准。

## 这次没有做的事

这次重构没有引入完整装备系统，也没有把选人 UI 改成无限动态卡片。现在的边界是：先把角色/武器实例化入口从 `main.gd` 中抽出来，降低后续新增内容时的主场景改动风险。
